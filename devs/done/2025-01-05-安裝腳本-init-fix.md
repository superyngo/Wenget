  現狀分析

  install.sh 目前做的事：

  6. 執行 wenget init --yes

  install.ps1 目前做的事：
  1. 偵測安裝模式（Admin/user）
  2. 下載 wenget binary
  3. 若 Admin，建立複本到 bin 目錄並修改系統 PATH
  4. 執行 wenget init --yes

  wenget init 目前做的事：
  1. 建立目錄結構（apps/bin/cache）
  2. 建立 manifests（installed.json, buckets.json）
  3. 設置 wenget 自身的 shim/symlink
  4. 修改 PATH（Windows registry 或 Unix shell config）
  5. 提示加入 wenget bucket

  ---
  問題與改進點

  | 項目     | 問題                                                      |
  |----------|-----------------------------------------------------------|
  | 重複邏輯 | install.ps1 在 Admin 模式下同時修改 PATH，與 init 重複    |
  | 缺少確認 | init 在修改系統前（PATH、shell config）沒有先提示用戶確認 |
  | 職責不清 | 安裝腳本做了太多事（偵測平台、設置 PATH 等）              |

  安裝腳本（install.sh / install.ps1）只做 3 件事：
  1. 下載 wenget binary 到 ~/.wenget/apps/wenget/ 或系統路徑
  2. 建立 shim/symlink 到 bin 目錄
  3. 執行 wenget init -y

  wenget init 增加互動確認：
  - 預設顯示將要修改的內容，等用戶確認
  - 支援 --yes/-y 跳過確認
